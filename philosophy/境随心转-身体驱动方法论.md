## 一、核心发现

### 1.1 身体先行，意识跟随

传统观念认为"先想清楚再行动"——这是死锁的来源。真正的机制与此相反：

```
身体状态 → 神经递质变化 → 意识状态切换 → 行为启动
```

不需要"下定决心"，不需要"痛改前非"。**身体里有一套已经被踩通的神经-血管回路**——每次执行，回路就加固一点。就像 SSH key 配对后无需密码即通。

### 1.2 "从一个世界转移到另一个世界"

第一次执行面部泵血协议后的真实体验：

- 变化极其清晰，但不是"轰"的一声
- 像**流水一样自然、不显眼**
- 上一个瞬间还感觉沉重无力，下一个瞬间能量已灌注
- 不是心理作用——是血液真实灌流到表皮毛细血管的生理结果

## 二、方法论：操作系统调度模型

### 2.1 死锁检测

每当感到"想做但动不了"，列出所有想做的事，找出互锁关系：

```
Process[A] = 想运动恢复  ───BLOCKED_BY──→ Resource[左膝] = 不可用
Process[B] = 想面部有活力  ───BLOCKED_BY──→ Process[A]     = 需要先运动
Process[C] = 想停止内耗    ───BLOCKED_BY──→ Process[A/B]   = 需要看到进展
                                        ↓
                              All processes waiting, none running.
                              Result: 躺着刷手机, 越躺越沉, 越沉越不想动.
```

**解法不是让它们同时启动，而是找一个无需任何前置条件的入口点。**

### 2.2 零依赖入口点

找到第一个**不需要膝盖、不需要意志力、不需要出门、不需要"下决心"**的执行路径。

问自己：什么动作能在 **2 分钟内从当前位置执行**？

### 2.3 Hook 绑定

把动作绑到必然发生的事件上：

| 触发点 | 动作 |
|--------|------|
| 双脚触地 | 踮脚尖 3 次 |
| 推开卫生间门 | 肩后旋 3 次 |
| 坐在椅子上感觉沉 | 挺直脊椎 10 秒 |
| 手机解锁但不知道干嘛 | 4-7-8 呼吸 1 次 |

不需要"从明天开始健身"。Hook 绑定 + 微习惯 = **流水般不显眼的转变**。

### 2.4 Watchdog 监控

用外部系统（PocketOS）监视你的微习惯状态：

```
pos ignite    # 90秒点火协议
pos check     # 身体 watchdog 检查
pos state     # 6维度状态捕获
pos history   # 查看状态变迁 + 长期趋势
```
